# Word 教案格式對應說明

本文件用來說明如何將 Skill 產出的 Markdown 課程設計內容，對應到 Word 教案格式。

---

## 一、對應原則

若使用者提供 Word 範本，應優先依照使用者範本中的欄位、表格與段落格式填入。

若使用者未提供 Word 範本，則可依照以下通用教案格式輸出。

---

## 二、通用 Word 教案欄位對應

| Word 欄位 | Skill 輸出區塊 |
| :--- | :--- |
| 課程名稱 | 課程基本資料表：課程名稱 |
| 適用年級 | 課程基本資料表：適用年級 |
| 總節數 | 課程基本資料表：總節數 |

| 議題融入 | 課程基本資料表：SDGs / 議題 |
| 設計理念 | 課程基本資料表：設計理念 |
| 課程大概念 | 課程架構分析：大概念 Big Idea |
| 核心素養 | 素養與學習重點：核心素養 |
| 學習表現 | 素養與學習重點：學習表現 |

---

## 三、Word 文件建議章節

建議 Word 文件依照以下順序排列：

1. 封面
2. 課程基本資料表
3. 課程設計理念
4. 課程架構分析
5. 素養與學習重點
6. K-U-D 學習目標
7. GRASPS 表現任務
8. 評量規準 Rubrics
9. 單元教學活動設計
10. 差異化與延伸建議
11. 附錄或學習單

---

## 四、產生 Word 檔時的注意事項

- 優先保留使用者提供的 Word 範本版面。
- 不任意更動原始表格欄位。
- 若欄位不足，可在適合的位置新增附表。
- 表格文字應避免過長。
- 教學活動流程應依節次與時間清楚排列。
- 若課綱代碼無法確認，應標示「待查證」或使用保守表述。

---

## 五、Markdown 到 Word 的轉換建議

可使用：

- 手動複製 Markdown 內容到 Word
- 使用 Pandoc 轉換
- 使用 Python `python-docx` 套件產生 `.docx`
- 使用既有 Word 範本進行欄位填入
