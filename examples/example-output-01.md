# 課程設計成果

## 一、課程基本資料表

| 項目 | 內容 |
| :--- | :--- |
| 課程名稱 | 從細胞到城市：用數據設計永續創新系統 |
| 適用年級 | 高中二年級 |
| 總節數 | 2 節課 |
| 跨領域科目 | 國文、生物、數學 |
| SDGs / 議題 | SDG 9 產業創新與基礎建設 |
| 課程大概念 | 自然界與人類社會中的系統，都需要透過結構、功能與回饋機制維持穩定與創新。 |
| 設計理念 | 本課程引導學生從生物系統的結構與功能出發，連結城市基礎建設與產業創新的運作邏輯，並透過數學資料分析與國文表達能力，提出一項具說服力的永續創新提案。學生不只是學習知識，而是練習以跨領域視角理解真實世界問題。 |

## 二、課程架構分析

### 1. 大概念 Big Idea

系統的穩定與創新，來自各部分之間的有效連結、資料回饋與功能調整。

### 2. 真實情境脈絡

城市交通、能源、水資源與產業發展，都像生物體一樣需要協調不同系統。當某一環節失衡，整體效能便會下降。學生將以「城市創新顧問」的角色，分析一項基礎建設問題，並提出改善方案。

### 3. 跨領域整合邏輯

| 領域 / 科目 | 在課程中的角色 | 與其他領域的連結 |
| :--- | :--- | :--- |
| 國文 | 建構提案論述、撰寫說服性文字 | 將生物與數學分析結果轉化為具說服力的倡議 |
| 生物 | 理解系統、結構、功能與回饋 | 作為理解城市系統與產業創新的類比基礎 |
| 數學 | 進行資料判讀、比例分析與圖表解讀 | 支持提案中的問題分析與改善依據 |

## 四、K-U-D 學習目標

| 類別 | 學習目標 |
| :--- | :--- |
| K：Know 知道 | 學生知道生物系統中的結構、功能與回饋概念；知道 SDG 9 的基本意涵；知道資料圖表可用來支持問題分析。 |
| U：Understand 理解 | 學生理解自然系統與社會系統都需要各部分協調運作；也理解創新不是單一發明，而是回應問題、改善系統的過程。 |
| D：Do 執行 | 學生能判讀簡易資料、建立系統類比、提出問題分析、撰寫說服性提案，並進行小組發表。 |

## 五、GRASPS 表現任務

| GRASPS 元素 | 說明 |
| :--- | :--- |
| G：Goal 目標 | 提出一項改善城市基礎建設或產業創新的永續提案。 |
| R：Role 角色 | 城市創新顧問。 |
| A：Audience 觀眾 | 市府代表、社區居民或青年創業團隊。 |
| S：Situation 情境 | 某城市面臨交通、能源、水資源或產業轉型問題，需要青年提出具可行性的創新方案。 |
| P：Product / Performance 產出或表現 | 小組完成一份一頁式提案，並進行 3 分鐘口頭發表。 |
| S：Standards 成功標準 | 提案需包含問題描述、資料依據、生物系統類比、數學分析、改善策略與說服性表達。 |

## 六、評量規準 Rubrics

| 評量向度 | 4 優秀 | 3 良好 | 2 基礎 | 1 待加強 |
| :--- | :--- | :--- | :--- | :--- |
| 概念理解 | 能精準說明系統、功能與回饋概念，並能遷移到城市或產業問題 | 能說明主要概念並合理應用 | 僅能說明部分概念，應用仍不完整 | 概念混淆，無法連結課程主題 |
| 跨域整合 | 能自然整合國文、生物與數學觀點 | 能整合兩個以上領域觀點 | 跨域連結較薄弱 | 只呈現單一科目內容 |
| 問題解決 | 提案具體、可行，且有資料支持 | 提案合理，有基本依據 | 提案方向可理解，但缺乏充分支持 | 提案模糊或不可行 |
| 表達溝通 | 文字與口語表達清楚、有說服力 | 表達清楚，能傳達重點 | 表達尚可，但組織略顯鬆散 | 表達不清，難以理解 |
| 合作參與 | 小組分工明確，能有效協作 | 多數成員能參與任務 | 參與程度不均 | 缺乏合作或任務完成度低 |

## 七、單元教學活動設計

| 節次 | 時間 | 教學流程 | 教師指導語 / 關鍵提問 | 學生活動 | 評量方式 / 工具 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 第一節 | 0-10 分鐘 | 引起動機 | 「如果一座城市是一個生命體，它的血管、神經與器官可能分別是什麼？」 | 學生自由聯想城市系統與生物體的相似處 | 口頭回應、便利貼 |
| 第一節 | 10-20 分鐘 | 概念建構 | 「生物系統如何透過不同構造合作維持生命？」 | 閱讀短文或圖片資料，整理生物系統概念 | 學習單 |
| 第一節 | 20-30 分鐘 | 跨域連結 | 「城市交通、能源或水資源系統，哪裡像生物系統？」 | 小組建立生物系統與城市系統的類比表 | 小組討論紀錄 |
| 第一節 | 30-50 分鐘 | 資料判讀 | 「這組資料透露了什麼問題？你如何用數據支持判斷？」 | 學生判讀簡易圖表，找出基礎建設問題 | 圖表分析單 |
| 第二節 | 0-10 分鐘 | 任務說明 | 「今天你們是城市創新顧問，要提出一個改善方案。」 | 理解 GRASPS 任務與評量規準 | 任務檢核表 |
| 第二節 | 10-25 分鐘 | 小組提案設計 | 「你的方案解決什麼問題？根據是什麼？」 | 小組完成一頁式提案草稿 | 教師巡迴回饋 |
| 第二節 | 25-40 分鐘 | 發表與回饋 | 「哪一組的方案最能兼顧創新、可行與永續？」 | 小組 3 分鐘發表，其他組給予回饋 | 同儕評量表 |
| 第二節 | 40-50 分鐘 | 反思收斂 | 「今天你如何用不同學科理解同一個問題？」 | 學生完成個人反思 | 出口票 |

## 八、差異化與延伸建議

### 1. 學習扶助

- 提供生物系統與城市系統的對照範例。
- 提供資料判讀句型，例如：「從圖表中我發現……因此我推測……」
- 提供一頁式提案模板，降低學生組織負荷。
- 小組內安排資料判讀員、文字整理員、發表員與時間管理員。

### 2. 充實挑戰

- 要求學生加入第二組資料作為佐證。
- 鼓勵學生比較不同城市案例。
- 增加成本、可行性或利害關係人分析。
- 將提案轉化為簡報、短影片或政策倡議書。
